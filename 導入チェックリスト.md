# 導入チェックリスト（お店ごとの設定）

このテンプレートを自分のお店用にするための手順です。**触るのは `schedule.json` と写真だけ**。
`src/` の中や `template.html` は触りません。詳しい画面手順は [運用ガイド.md](./運用ガイド.md) を参照。

---

## 1. テンプレートを自分用にコピー

- GitHub のこのリポジトリで「**Use this template**」→ 自分のアカウントに **Public** で作成。
- パソコンに GitHub Desktop で取り込む（[運用ガイド.md](./運用ガイド.md) 参照）。

## 2. お店の情報を書く（`schedule.json`）

このファイルだけ編集します。編集したら GitHub Desktop で Commit → Push。

- [ ] `shopName` … お店の名前
- [ ] `location` … 地域（例: `Denenchofu, Tokyo`）
- [ ] `regular` … 曜日ごとの営業時間。休みの曜日は `null`
  ```json
  "regular": {
    "mon": null,
    "tue": { "open": "12:00", "close": "18:00" },
    ...
  }
  ```
- [ ] `theme.accent` … 差し色（OPENの時間・CLOSEDの文字・区切り線の色）。例 `#0abab5`
- [ ] `postClosedDays` … `false` 推奨（定休日は投稿しない／臨時休業は投稿する）

## 3. 臨時休業用のスプレッドシートを作る

- [ ] Googleスプレッドシートを新規作成、1行目に `日付 / 営業時間 / 備考`。
- [ ] 日付の列を「書式なしテキスト」に。
- [ ] ファイル → 共有 → ウェブに公開 → **CSV** → URLをコピー。
- [ ] `schedule.json` の `overridesSheetCsvUrl` にそのURLを貼る。

（使い方: 臨時休業=日付だけ＋営業時間空欄 / 時間変更=`14:00-18:00` / 備考=一言）

## 4. 背景写真を入れる

- [ ] `backgrounds/` フォルダに写真を数枚（jpg/png/heic）。日替わりで自動的に使われます。
- [ ] 中央がごちゃつかない・やや暗めの写真だと文字がきれいに乗ります。

## 5. Instagram と連携（一番のヤマ場）

[運用ガイド.md](./運用ガイド.md) の「方式B」を見ながら進めます。要点だけ:

- [ ] Instagram をプロアカウントにして Facebook ページと連携。
- [ ] Meta でアプリ作成（ユースケース「Instagramでメッセージとコンテンツを管理」）。
- [ ] 自分のInstagramをテスター登録＆承認 → トークンを生成。
- [ ] `user_id`（IG_USER_ID）を取得。
- [ ] トークン自動更新用に PAT（`GH_PAT`）を用意。

## 6. GitHub に鍵を登録

リポジトリ → Settings → Secrets and variables → Actions で登録:

- [ ] `IG_USER_ID`
- [ ] `IG_ACCESS_TOKEN`
- [ ] `GH_PAT`（トークン自動更新用）

## 7. 動作確認

- [ ] Actions タブ →「daily-instagram-story」→ Run workflow でテスト投稿できるか確認。
- [ ] （アカウント）Settings → Notifications → Actions で**失敗通知をオン**（推奨）。
- [ ] トークン自動更新のテスト：Actions →「refresh-instagram-token」→ Run workflow が成功するか一度確認（失敗すると60日で投稿が止まる）。

## 8. 正確な時刻で投稿する（cron-job.org）※推奨

GitHubの定時は数時間ズレることがあるので、時刻をきっちり出すなら外部cronから叩く。
詳しい手順は [運用ガイド.md](./運用ガイド.md) の「正確な時刻で投稿する（cron-job.org）」参照。

- [ ] dispatch用トークン作成（Fine-grained、**Actions: Read and write**、対象リポのみ）。
- [ ] cron-job.org に登録（URL / POST / Headers / body `{"ref":"main"}` / Asia/Tokyo / 9:30）。TEST RUNで**204**を確認。
- [ ] 予備トリガーを Clone で2つ（9:33 / 9:36、複製は無効状態なので**Enableをオン**に）。
- [ ] GitHub側の `daily-story.yml` の `cron` は 9:40 目標のバックアップとして残す。

---

完了すると、毎朝その日の営業案内が正確な時刻に自動投稿され、臨時休業はスプレッドシートに書くだけで反映されます。
