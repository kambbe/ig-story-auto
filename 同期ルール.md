# 同期ルール（本番リポ ⇄ 配布用テンプレ）

2つのリポがある：
- **本番（開発用）** … 自分のお店用。ここで改良を試す。
- **配布用テンプレ** … 他店に配る雛形（`ig-story-auto-template`）。

改良は本番で試して、良ければ配布用にも同じ変更をコピーして「ベースを一致」させる。
設定（お店ごとの中身）は同期しない。以下の表がすべて。

---

## ✅ 同期する（＝共通のベース。両方を必ず一致させる）

ここを本番でいじって良ければ、**同じファイルを配布用の同じ場所に上書きコピー**する。

| ファイル | 中身 |
|----------|------|
| `src/status.mjs` | 営業/休業の判定ロジック |
| `src/render.mjs` | 画像生成・シート読み込み・背景処理 |
| `src/post.mjs` | Instagram投稿 |
| `src/refresh-token.mjs` | トークン更新 |
| `src/template.html` | ストーリーの見た目（配色は変数で調整） |
| `.github/workflows/daily-story.yml` | 毎朝の投稿ワークフロー |
| `.github/workflows/refresh-token.yml` | トークン自動更新ワークフロー |
| `package.json` / `.gitignore` | 依存・除外設定 |
| `運用ガイド.md` / `導入チェックリスト.md` | 手順書 |

## ❌ 同期しない（＝お店ごとの設定。各リポで別々でOK）

| もの | 理由 |
|------|------|
| `schedule.json` | 店名・営業時間・シートURL・色・overrides はお店ごとに違う |
| `backgrounds/` の写真 | お店ごとの写真 |
| GitHub の Secrets（`IG_USER_ID` / `IG_ACCESS_TOKEN` / `GH_PAT`） | お店ごとのアカウント。そもそもコードには入らない |
| `README.md` | 中身は共通だが、配布用だけ冒頭に「Use this template」の案内を足している |

---

## 変更を配布用に反映する手順

1. 本番で `src/` などベースのファイルを直して、動作OKを確認（Actionsで Run workflow）。
2. 直したファイルを、配布用テンプレの**同じパスに上書きコピー**。
3. 配布用でコミット＆プッシュ。
4. `schedule.json` と `backgrounds/` は触らない（設定なので別々のまま）。

## 見た目・色を変えたいとき

`src/template.html` の CSS を直す（＝ベースの変更なので両方に反映）。
ただし **色だけ**なら各店が `schedule.json` の `theme.accent` で変えられるので、
テンプレ側のCSSは触らなくてよい（色はお店ごとの設定として分離済み）。
