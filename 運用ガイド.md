# 運用ガイド

画像を毎日自動生成するところは完成済み。残りは「その画像をどう投稿するか」の2択です。

---

## 方式A：半自動（おすすめ・設定が楽）

毎朝GitHub Actionsがその日の画像を作り、**Gmailに送信**。スマホで受け取ってストーリーに投稿します。
Meta審査もビジネス認証も不要。リポジトリは非公開でOK。リンクスタンプも手動で貼れます。

### 初期セットアップ（1回だけ）

1. **GitHubにリポジトリを作成**（非公開でOK）し、このフォルダをpush。
2. **Gmailのアプリパスワードを取得**
   - Googleアカウント → セキュリティ → 2段階認証をON → 「アプリパスワード」を発行（16桁）。
3. リポジトリの **Settings → Secrets and variables → Actions** に登録：
   - `MAIL_USERNAME` … あなたのGmailアドレス
   - `MAIL_PASSWORD` … 発行した16桁のアプリパスワード
4. 半自動用ワークフロー（`.github/workflows/daily-mail.yml`）を置く（別途用意します）。

### 毎日の流れ

- 朝、届いたメールの画像を長押しで保存 → Instagram → ストーリーに追加 → 投稿。
- リンクを貼りたい日は、投稿画面で「リンクスタンプ」を貼るだけ（オンラインストアやShopify商品へ）。

### 変更したいとき

- **臨時休業・時間変更**：`schedule.json` の `overrides` に日付を足してpush。
- **写真を追加**：`backgrounds/` に画像を足してpush（HEICのままでOK）。

## 方式B：完全自動（API投稿・無人）

毎朝GitHub ActionsがInstagram APIで自動投稿。**自分の店のアカウントに投稿するだけなら、App Review（審査）もビジネス認証も不要**で、アプリは「開発モード（Standard Access）」のまま使えます。

### 初期セットアップ（1回だけ・順番に）

**① Instagram側**
- Instagramを**プロアカウント**（ビジネス or クリエイター）にする。
- **Facebookページ**を1つ作り、そのIGアカウントと連携（IGアプリ設定 → ページとリンク）。

**② Metaアプリを作る**
- [developers.facebook.com](https://developers.facebook.com/) にログイン → My Apps → **Create App**。
- タイプは「ビジネス」。作成後、プロダクトから **Instagram Graph API** を追加。
- 自分がアプリの管理者なので、開発モードのまま自分のアカウントを操作できる。

**③ トークンとIDを取る（Graph API Explorer）**
- アプリ管理画面 → Tools → **Graph API Explorer**。
- アクセス許可（Permissions）で次を追加：
  `instagram_basic` / `instagram_content_publish` / `pages_show_list` / `pages_read_engagement`
- 「Generate Access Token」→ 自分のFacebook・ページ・IGを許可 → **短期トークン**が出る。
- 次のリクエストで **IGユーザーID** を取得：
  `me/accounts?fields=instagram_business_account`
  → 返ってきた `instagram_business_account.id`（数字）が `IG_USER_ID`。

**④ 短期トークンを長期トークン（約60日）に交換**
- `src/refresh-token.mjs` を使うか、Explorerの「Access Token Debugger」からも延長可。
- 必要な値：`FB_APP_ID` / `FB_APP_SECRET`（アプリの Settings → Basic にある）。
  ```bash
  FB_APP_ID=xxx FB_APP_SECRET=yyy IG_ACCESS_TOKEN=短期トークン node src/refresh-token.mjs
  ```
  → 出てきた長期トークンが `IG_ACCESS_TOKEN`。

**⑤ GitHubに載せる**
- このフォルダを **公開（Public）** リポジトリでpush（画像を公開URLで配信するため。営業案内画像なので公開でOK）。
- Settings → Secrets and variables → Actions に登録：
  `IG_USER_ID` / `IG_ACCESS_TOKEN`
- 既存の `.github/workflows/daily-story.yml` が毎朝 JST 08:00 に自動投稿。
- まず Actions タブの「Run workflow」で手動実行し、テスト投稿できるか確認。

### 運用の注意

- **トークンは約60日で失効** → `node src/refresh-token.mjs` で再発行し、Secretを更新。
  （60日ごとが面倒なら、月1でトークンを自動リフレッシュするワークフローも追加できます。）
- ストーリーへの**リンクスタンプはAPIでは付けられない**（自動投稿では貼れない）。
- 1日25投稿まで。ストーリーは24時間で消える。

### 変更したいとき

- 方式Aと同じ。`schedule.json` と `backgrounds/` を編集してpushするだけ。

## どちらを選ぶ？

| | 半自動（A） | 完全自動（B） |
|---|---|---|
| 初期設定 | 楽（メール設定のみ） | 重い（Meta審査・認証） |
| 毎日の手間 | スマホで数タップ | なし |
| リンクスタンプ | 貼れる | 貼れない |
| リポジトリ | 非公開OK | 公開が必要 |

小さな店で「オンライン導線も貼りたい」なら **方式A（半自動）が相性◎**。
「とにかく無人がいい」なら方式B。まずAで始めて、慣れたらBに移行するのも手です。

投稿時刻は各ワークフローの `cron`（UTC）で調整できます（JST 08:00 = `0 23 * * *`）。

## スマホから臨時休業を追加（Googleスプレッドシート方式）

`schedule.json` を触らず、スプレッドシートに行を足すだけで臨時休業/時間変更を反映できます。
毎朝のレンダリングがシートを読みに行くので、GitHubは触りません（翌朝から反映）。

### 使い方（毎回）

スプレッドシートは **2列だけ**。1行足すだけです：

| 日付 | 営業時間 |
|------|----------|
| 2026-07-20 | 14:00-18:00 |
| 2026-07-22 |  |

- **臨時休業** … 日付だけ入れて、営業時間は空欄。
- **時間変更** … 営業時間に `14:00-18:00`（`14:00〜18:00` でもOK）。

### 初期設定（1回だけ）

1. Googleスプレッドシートを作り、1行目に見出し `日付 / 営業時間`。
2. **日付の列を「書式なしテキスト」に**（列を選択 → 表示形式 → 数字 → 書式なしテキスト）。
   これをしないと日付が勝手に変換される。
3. ファイル → 共有 → **ウェブに公開** → シートを選び **CSV** 形式で公開 → URLをコピー。
4. `schedule.json` の `"overridesSheetCsvUrl": ""` にそのURLを貼って、コミット＆プッシュ（1回だけ）。

以後はシートを編集するだけ。同じ日付はシートがJSONの `overrides` より優先される。
シート取得に失敗しても、JSONの `overrides` で投稿は継続する（止まらない）。
